# OPUS RPS AI - Championship-Level Rock Paper Scissors AI

## Обзор

OPUS AI - это продвинутый алгоритм для игры в "Камень-Ножницы-Бумага", использующий **ансамблевое предсказание** с динамической корректировкой весов на основе исторической точности каждой стратегии.

## Ключевые особенности

- **13 стратегий предсказания** работающих параллельно
- **Динамические веса** - стратегии с лучшей точностью получают больший вес
- **Память по игрокам** - хранит историю до 2 дней с момента последнего обновления
- **Anti-exploitation** - случайная вариация для защиты от предсказуемости
- **Подробный debug-режим** для анализа работы алгоритма

## Стратегии предсказания

### 1. Frequency Analysis (частотный анализ)

Анализирует общую частоту выбора каждого хода противником. Использует сглаживание Лапласа для избежания нулевых вероятностей.

### 2-5. Markov Chains (Марковские цепи) порядков 1-4

- **Markov-1**: P(следующий | последний ход)
- **Markov-2**: P(следующий | последние 2 хода)
- **Markov-3**: P(следующий | последние 3 хода)
- **Markov-4**: P(следующий | последние 4 хода)

Более высокий порядок = более специфичные паттерны, но требует больше данных.

### 6. Meta-Strategy (мета-стратегия)

Анализирует поведение противника после победы/поражения/ничьей:

- После победы люди часто повторяют ход
- После поражения часто меняют стратегию

### 7. Transition Analysis (анализ переходов)

Матрица переходов: какой ход обычно следует за текущим.

### 8. Pattern Matching (LZ-style)

Ищет длинные повторяющиеся паттерны (до 6 ходов) в истории и предсказывает следующий ход.

### 9. Anti-Repetition (анти-повторение)

Люди избегают повторять один ход много раз подряд. Если противник повторил ход, вероятность третьего повтора снижается.

### 10. Win-Stay-Lose-Shift (WSLS)

Классическая стратегия:

- Выиграл → повтори ход
- Проиграл → смени на контр-ход к ходу противника

### 11. Beat-Last-Move

Люди часто пытаются побить последний ход противника.

### 12. Game Pattern (паттерн текущей игры)

Анализирует частоты только в рамках текущей партии (до 10 раундов).

### 13. Cycle Detection (детекция циклов)

Обнаруживает циклические паттерны:

- Прямой цикл: R→P→S→R
- Обратный цикл: R→S→P→R

## Ансамблевое предсказание

### Формула веса стратегии:

```
weight = base_weight × confidence

base_weight = max(0.1, (accuracy - 0.33) × 6 + 1)
```

Где:

- `accuracy` = 70% недавняя точность + 30% общая точность
- `confidence` = уверенность стратегии (0-1)

### Комбинирование предсказаний:

```
P(move) = Σ(strategy_weight × strategy_probability) / Σ(strategy_weight)
```

### Выбор хода:

Вычисляется **Expected Value (EV)** для каждого хода:

```
EV(our_move) = Σ P(enemy_move) × outcome_value
```

Где outcome_value: win=+1, draw=0, loss=-1

## Anti-Exploitation

С вероятностью 15% выбирается случайный ход среди "хороших" (EV в пределах 0.1 от лучшего). Это защищает от предсказуемости.

## Управление памятью

- **Хранение**: 2 дня с момента последнего обновления
- **Очистка**: каждый час проверяются и удаляются устаревшие записи
- **Структура**: отдельная запись для каждого логина игрока

Установите `IS_DEBUG = true` для вывода:

- Expected Values для каждого хода

## Теоретическое обоснование

Алгоритм основан на исследованиях в области:

- **Теория игр**: Nash equilibrium в RPS = 1/3 для каждого хода
- **Поведенческая экономика**: люди не играют случайно
- **Machine Learning**: ансамблевые методы превосходят отдельные модели
- **Соревновательное программирование**: лучшие RPS боты используют похожие подходы

Против идеального случайного противника OPUS сходится к 50% побед.
Против людей с паттернами - значительно выше.
